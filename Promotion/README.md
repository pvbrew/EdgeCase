# Promotion drafts for 1.0

Ready-to-post drafts for the v1.0 announcement. Nothing here publishes
itself — each is a manual step:

- [ ] **Swift Forums** — post [swift-forums.md](swift-forums.md) in the
  [Related Projects](https://forums.swift.org/c/related-projects) category.
- [ ] **r/swift** — post [reddit-r-swift.md](reddit-r-swift.md) (title
  suggestion included at the top of the file).
- [ ] **dev.to / Medium** — publish [article-dev-to.md](article-dev-to.md)
  (dev.to front matter included; strip it for Medium).
- [ ] **Swift Package Index** — submit the repo at
  [swiftpackageindex.com/add-a-package](https://swiftpackageindex.com/add-a-package)
  (`.spi.yml` already requests docs builds for all three modules).
- [ ] **GitHub Pages** — enable Pages (Settings ▸ Pages ▸ Source: GitHub
  Actions) so the Docs workflow can publish on the `1.0.0` tag.

Before posting, sanity-check that the hosted docs URL
(https://pvbrew.github.io/EdgeCase/) resolves — the drafts link to it.
